Din superenkla fusklapp: GitHub + Dubbletter

Skriven så enkelt som möjligt. Tänk så här: du har en leksakslåda (projektet), och GitHub är din extra trygga låda på internet.

Tips: Öppna terminal i VS Code med Ctrl+ö.

VARJE GÅNG DU HAR ÄNDRAT KOD

Skicka upp nya ändringar till GitHub

  1. Samla alla ändringar.
  2. Skriv en kort text om vad du gjorde.
  3. Skicka upp till GitHub.
git add . git commit -m "Min ändring" git push

Byt texten "Min ändring" till vad du faktiskt gjorde.

OM DET BLIR FEL (EXIT CODE 1)

Vad hände nyss? Enkel förklaring

Du gjorde inget farligt. Din commit blev sparad lokalt, men Git visste inte vart den skulle skickas första gången på denna branch.

Lösning: koppla branchen till GitHub en gång. Sen räcker vanligt git push.

git push --set-upstream origin main

Efter detta: kör vanligt flöde igen: git add . + git commit -m "..." + git push.

KOLLA DUBLETTER

Se om samma pizza_namn finns flera gånger i samma pizzeria

node scripts/hitta-dubletter.js

Detta uppdaterar också rapporten: scripts/dubletter-rapport.html

TA BORT DUBLETTER AUTOMATISKT

Auto-fix av dubbletter

Viktigt: Scriptet behåller första posten och tar bort resten.

node scripts/hitta-dubletter.js --fixa

Tryggt: backup-fil skapas automatiskt i data-mappen.

ÖPPNA RAPPORTEN DIREKT

Visa rapporten i webbläsaren

start scripts/dubletter-rapport.html
OM DU VILL ÅNGRA EN AUTO-FIX

Byt tillbaka till senaste backup

1) Se backup-filer. 2) Kopiera filnamnet. 3) Ersätt pizzor.json med backup-filen.

Get-ChildItem data\*.backup-*.json | Sort-Object LastWriteTime -Descending | Select-Object -First 5 Name, LastWriteTime
Copy-Item "data\pizzor.json.backup-XXXXXXXXXXXX.json" "data\pizzor.json" -Force

Byt XXXXXXXXXXXX till riktig sifferserie i backup-filens namn.

LITEN MINNESREGEL

Som för en 5-åring

  1. Du bygger sandslott (kodar lokalt).
  2. Du tar foto på sandslottet (git commit).
  3. Du lägger fotot i molnlåda (git push till GitHub).
  4. Om sandslottet blir konstigt: titta i rapporten och fixa.
