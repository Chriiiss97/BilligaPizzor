Din superenkla fusklapp: GitHub + Dubbletter

Skriven sa enkelt som mojligt. Tank sa har: du har en leksakslada (projektet), och GitHub ar din extra trygg lada pa internet.

Tips: Oppna terminal i VS Code med Ctrl+o (knappen med o/oe pa svenskt tangentbord).

VARJE GANG DU HAR ANDRAT KOD

Skicka upp nya andringar till GitHub

  1. Samla alla andringar.
  2. Skriv en kort text om vad du gjorde.
  3. Skicka upp till GitHub.
git add . git commit -m "Min andring" git push

Byt texten "Min andring" till vad du faktiskt gjorde.

KOLLA DUBLETTER

Se om samma pizza_namn finns flera ganger i samma pizzeria

node scripts/hitta-dubletter.js

Detta uppdaterar ocksa rapporten: scripts/dubletter-rapport.html

TA BORT DUBLETTER AUTOMATISKT

Auto-fix av dubbletter

Viktigt: Scriptet behaller forsta posten och tar bort resten.

node scripts/hitta-dubletter.js --fixa

Tryggt: backup-fil skapas automatiskt i data-mappen.

OPPNA RAPPORTEN DIREKT

Visa rapporten i webblasaren

start scripts/dubletter-rapport.html
OM DU VILL ANGRA EN AUTO-FIX

Byt tillbaka till senaste backup

1) Se backup-filer. 2) Kopiera filnamnet. 3) Ersatt pizzor.json med backup-filen.

Get-ChildItem data\*.backup-*.json | Sort-Object LastWriteTime -Descending | Select-Object -First 5 Name, LastWriteTime
Copy-Item "data\pizzor.json.backup-XXXXXXXXXXXX.json" "data\pizzor.json" -Force

Byt XXXXXXXXXXXX till riktig sifferserie i backup-filens namn.

LITEN MINNESREGEL

Som for en 5-aring

  1. Du bygger sandslott (kodar lokalt).
  2. Du tar foto pa sandslottet (git commit).
  3. Du lagger fotot i molnlada (git push till GitHub).
  4. Om sandslottet blir konstigt: titta i rapporten och fixa.
